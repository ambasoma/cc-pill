# cc-pill

A living menu bar for [Claude Code](https://claude.com/claude-code). A Dynamic Island style pill floats over your macOS menu bar showing every Claude Code session on the machine, a pair of tiny cats animate while Claude works, a paw taps the screen when Claude needs you, and a local neural voice speaks a short, personality-driven briefing when each turn finishes. Press a hotkey, ask for something in plain language, and a hands-free Claude session spins up in the background and reports back out loud.

Built for people who run Claude Code while doing other things.

## What it does

- **Session island.** A slim pill at the top-center of the screen, porcelain by day and dark-kiln by night (follows system appearance, sun and crescent-moon glyphs). Hidden when nothing is running; its appearance IS the signal.
- **Live narration.** While Claude works, the pill alternates between the project name and what Claude is doing right now ("editing worker.py", "running the tests", "using GitHub"), with elapsed minutes on long turns. Two tiny cats take laps; at night a kiln ember breathes behind them.
- **Needs-you paw.** Real permission prompts get an amber ping and a paw tapping the screen from under the pill. Idle nags are filtered out.
- **Spoken briefings.** When a turn ends, the pill blooms open and the words fill in, paced to the audio, while your named assistant (default: Jarvis) speaks a 1-3 sentence summary. Never a transcript: code, paths, and IDs are described in plain speech, and the persona is yours to choose or generate. Local TTS ([Kokoro](https://github.com/thewh1teagle/kokoro-onnx)), summarized by Haiku through your existing Claude Code login. If your music is playing, it ducks, speaks, and resumes.
- **Courtesy layer.** Silent (text-only bloom) when you are already looking at that session's terminal. Silent for SSH sessions. Re-orients you ("On the api work, ...") when the last briefing you heard was a while ago or from another project. Turn-end critter moments: a star pop for good news, ears-back slow walk for failures.
- **The card.** Hover the pill to peek, click to pin: one row per session with state and elapsed time, plus replay-briefing, speak-a-prompt (native speech recognition with a text fallback), and open-terminal buttons.
- **Ask mode.** `⌥⌘M` anywhere: the pill becomes a prompt. Speak or type, hit Enter, and a detached Claude session starts in your configured directory (auto-permission mode, optional extra system instructions), works hands-free, and briefs you out loud when done. Idle pill-launched sessions are garbage collected after 30 minutes.

## Requirements

- macOS 14+ on Apple Silicon
- [Claude Code](https://claude.com/claude-code) installed and logged in
- Xcode command line tools (`xcode-select --install`)
- Homebrew (for `tmux` and `jq`; `media-control` optional for audio ducking)

No Xcode project, no code signing, no accounts. Everything runs locally; the only network calls are your own Claude Code sessions and the one-time model download (~350MB).

## Install

```
git clone https://github.com/ambasoma/cc-pill
cd cc-pill
./setup.sh
```

The interactive setup asks where to install, which voice (with live previews), your assistant's name, its personality (presets, your own text, or described-and-generated by Haiku), and how pill-launched sessions should run. It then builds the app, installs a launchd agent (starts at login, auto-restarts if killed), registers the Claude Code hooks globally, and optionally adds a tmux wrapper for your interactive sessions so the pill can talk to them.

## Controls

- `⌥⌘M` — ask mode: start a new session by voice or text (press again to send, Esc to cancel)
- Hover the pill — peek the session card; click to pin
- `<yourname>` in the shell (alias created by setup): `mute`, `on`, `stop`, `last` (replay briefing), `say <text>`, `voice <name>`, `vol`, `speed`
- Config: `~/.cc-pill/config.json` (name, voice, working dir, auto mode, GC window, extra system prompt)
- Logs: `~/.cc-pill/pill.log` (the app narrates everything it does) and `voice/spool/voice.log`

## How it works

```
every Claude Code session
  -> global hooks (hooks/hook.py) write tiny JSON events to ~/.cc-pill/events/
  -> Pill.app (Swift, a fixed non-resizing panel above the menu bar with
     per-pixel mouse pass-through) watches the directory and animates
turn ends
  -> voice/hook.py -> worker: claude -p (Haiku) rewrites the final message
     into a spoken line -> Kokoro synthesizes -> the app blooms in sync
ask mode
  -> the app spawns claude in a detached tmux session and delivers your
     prompt once the TUI is ready; hooks pick it up like any session
```

## Uninstall

```
launchctl bootout gui/$(id -u)/com.ccpill.pill
rm -rf ~/.cc-pill ~/Library/LaunchAgents/com.ccpill.pill.plist
```

Then remove the cc-pill hooks block from `~/.claude/settings.json`, the wrapper/alias lines from `~/.zshrc`, and the install folder.
